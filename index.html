Curso Básico de Git y GitHub

1. ¿Qué es Git y GitHub?

Git

Git es un sistema de control de versiones distribuido que permite:

  • Registrar cambios en archivos
  • Trabajar en equipo de forma coordinada
  • Revertir cambios cuando sea necesario
  • Crear ramas para desarrollar características independientes

GitHub

GitHub es una plataforma de alojamiento para proyectos que usan Git. Ofrece:

  • Almacenamiento en la nube de repositorios
  • Herramientas para colaboración
  • Funcionalidades sociales (seguir proyectos, dar estrellas)
  • GitHub Pages para alojar sitios web

2. Configuración inicial

Instalación

Descarga Git desde git-scm.com e instálalo en tu sistema.

Configuración básica

Comandos básicos de configuración:

git config --global user.name "Tu Nombre"
git config --global user.email "tu@email.com"
git config --global core.editor "code --wait"

3. Conceptos básicos

Repositorio

Directorio donde Git realiza el seguimiento de los cambios en los archivos.

Commit

Un "snapshot" o instantánea de los cambios en un momento determinado.

Rama (Branch)

Línea de desarrollo independiente que permite trabajar en características sin afectar la versión principal.

Clonar (Clone)

Copiar un repositorio remoto a tu máquina local.

4. Comandos básicos de Git

Iniciar un repositorio

git init

Crea un nuevo repositorio Git en el directorio actual.

Ver el estado

git status

Muestra el estado del directorio de trabajo y del área de preparación.

Añadir cambios

git add nombre_archivo
git add .

Añade archivos al área de preparación (staging area).

Confirmar cambios

git commit -m "Mensaje descriptivo"

Crea un commit con los cambios preparados.

Ver historial

git log
git log --oneline

Muestra el historial de commits.

5. Trabajando con ramas

Crear una rama

git branch nombre_rama

Cambiar de rama

git checkout nombre_rama
git switch nombre_rama

Crear y cambiar a nueva rama

git checkout -b nombre_rama

Fusionar ramas

git merge nombre_rama

Fusiona los cambios de la rama especificada con la rama actual.

Eliminar rama

git branch -d nombre_rama

6. Trabajando con GitHub

Clonar un repositorio

git clone https://github.com/usuario/repositorio.git

Conectar repositorio local a remoto

git remote add origin https://github.com/usuario/repositorio.git

Subir cambios (push)

git push -u origin main

Sube los commits locales al repositorio remoto.

Actualizar repositorio local (pull)

git pull origin main

Descarga los cambios del repositorio remoto y los fusiona con tu versión local.

Fork y Pull Request

Para contribuir a proyectos ajenos:

  1. Haz un fork del proyecto en GitHub
  2. Clona tu fork localmente
  3. Crea una rama para tus cambios
  4. Haz commits y push a tu fork
  5. Crea un Pull Request desde tu fork al repositorio original

7. Flujo de trabajo básico

Ejemplo de flujo diario:

git pull origin main
git checkout -b nueva-funcionalidad

# Realiza cambios en los archivos...

git add .
git commit -m "Agrega nueva funcionalidad"
git push origin nueva-funcionalidad

# Luego en GitHub: crear Pull Request y esperar revisión

8. Resolviendo conflictos

Cuando Git no puede fusionar cambios automáticamente:

  1. Abre los archivos con conflictos (marcados con <<<< y >>>>)
  2. Edita para resolver las diferencias
  3. Marca como resuelto:
    4. git add archivo_con_conflicto
  4. Completa la fusión:
    5. git commit

9. Buenas prácticas

  • Haz commits atómicos (un cambio por commit)
  • Escribe mensajes de commit claros y descriptivos
  • Haz pull frecuentemente para mantenerte actualizado
  • Usa ramas para cada nueva característica o corrección
  • Ignora archivos innecesarios con .gitignore

Archivo .gitignore

Especifica archivos que Git debe ignorar:

# Ejemplo de .gitignore
node_modules/
*.log
.DS_Store
.env
